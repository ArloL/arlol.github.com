1.1 Motivation

Why am I doing this?

The number of electronic devices used in everyday and professional life has and will continue to increase. This rise is driven mostly by the widespread adoption of smartphones and tablets. The trend towards technological ubiquity will continue with the adoption of e.g. tabletops, public touch displays and in-car computers. This rising number of devices creates challenges and opportunities for the design of applications. One of the opportunities is the ability to create applications that interact with each other across devices. The possibilities reach from basic content-sharing like images to complex interactions that include gesture recognition and location tracking. Since many of the possible applications (see chapter Application Scenarios) have common requirements it makes sense to create a single framework that enables Device Interaction [DI]. This allows developers to focus on implementing the interactions and not on details like network communication and device discovery. Despite the abstraction it should nonetheless be easy for a developer to understand the technologies - most importantly for the communication - used in such a framework. Given the ubiquity of the Internet it is highly likely that said developer has created an application that communicates using web technologies (e.g. HTTP, REST, JSON). Therefore such a framework should use web technologies for the communication. This utilises the existing knowledge in the developer community and has the same advantages that gave web technologies the ability to be used broadly. As for particular technologies the architecture style REST should be used due to popularity and applicability

This thesis will analyze the requirements of such a framework and then implement it.

1.3 Research questions

This thesis presents a new approach for device communication that allows interacting applications.

  1. Is it possible to have web servers on mobile devices?
  2. Is the performance sufficient?

1.4 Goals

There are two main goals of this thesis. The first is to develop a framework that enables Device Interaction using web technologies. The second goal is to attempt to answer the research questions listed above to show that the approach followed while developing the framework is feasible for creating interacting applications.

1.5 Conclusion

In this chapter an approach for Device Interaction using web technologies is presented. Device Interaction in general was introduced and basic requirements of communication approaches were identified. The problems of existing communication approaches preventing Device Interaction have been identified. Finally, the research questions that will be investigated and the goals of this thesis were stated.
